### Task 2: AudioService core — state, two-level mute, optional persistence, host, Dispose

### Task 3: SfxPlayer — pooled one-shots, variations, pitch, retrigger throttle

Internal class owned by the service. `Play(Entry, float extraScale)` and direct `Play(AudioClip, float volumeScale, float pitch)`: skip when `EffectiveSfxVolume <= 0`; throttle catalog keys via `Time.unscaledTime` map (spec §9); random clip pick; pitch `1 ± variance`; configure pooled source (volume = effective × scales, pitch), `Play()`, async unscaled return-timer (`clip.length / pitch`) guarded by dispose/source-alive; active-shot list `{source, baseScale}` so `ApplyVolumes` retro-applies mute/volume to in-flight shots (ad-mute silences immediately). Addressable-only Sfx entry → error + skip (spec §6). Tests: disabled/muted spawns nothing (host child count), throttle drops rapid re-play, direct overload bypasses throttle, pitch within bounds, active-shot volume updates on PushMute.

- [ ] Implement + tests + commit `feat(audio): pooled sfx with variations and retrigger throttle`.

### Task 4: MusicPlayer — crossfade, playlists, addressable pipeline

Internal class, two slots `{source, handle?, fadeWeight, entryScale, key}`. Public service API: `PlayMusic(string key, bool loop = true)`, `PlayMusic(AudioClip, bool loop = true)`, `PlayPlaylist(string key)`, `StopMusic()`, `MusicCrossfadeSeconds` (default 0.5), `ActiveMusicKey`, `ActivePlaylistKey`. Generation-guarded async flows per spec §7–8: idempotence checks before bumping; addressable load inside the pipeline (release on abandon); crossfade via per-slot fadeWeight × `EffectiveMusicVolume` × entryScale (volume gating keeps position); playlist order shuffled once per start; advance monitor fires when `remaining <= max(crossfade, ~0.05)`; list end → loop or stop; handles released when a slot drops its clip. Tests (state-level, loops park in EditMode): PlayMusic assigns clip+key, idempotent re-call keeps generation state, PlayPlaylist sets `ActivePlaylistKey` + first track, shuffle yields permutation of entryKeys, StopMusic clears keys, unknown keys error+no-op, non-Music entry rejected for playlists.

- [ ] Implement + tests + commit `feat(audio): crossfading music player with playlists and addressable clips`.
