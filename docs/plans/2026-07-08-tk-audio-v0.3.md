### Task 1: Mute redesign core — MusicPlayVolume, pause, PauseMusic/ResumeMusic

**Files:** `Runtime/AudioService.cs`, `Runtime/MusicPlayer.cs`, `Runtime/Audio.cs`, `Tests/Editor/AudioServiceTests.cs` (update the one changed test + add pause tests).

- **AudioService volume model:** add `private float _musicFade = 1f, _sfxFade = 1f;` and `private bool _musicManuallyPaused;`. Replace `EffectiveMusicVolume` with `internal float MusicPlayVolume => _data.MusicVolume * _musicFade;` (NO enabled/mute gate). Change `EffectiveSfxVolume` to `(_data.Sfx && !IsMuted ? _data.SfxVolume : 0f) * _sfxFade`. Add `internal bool MusicPaused => IsMuted || _musicManuallyPaused;`.
- **PushMute/PopMute:** replace the `ApplyVolumes()` call with `OnMuteChanged()` → `_music.SetPaused(MusicPaused); _sfx.ApplyVolumes();`. (SFX still volume-gates on mute; music now pauses.)
- **PauseMusic/ResumeMusic:** set `_musicManuallyPaused` then `_music.SetPaused(MusicPaused)`.
- **MusicPlayer:** every `_owner.EffectiveMusicVolume` → `_owner.MusicPlayVolume`. Add `public void SetPaused(bool paused)` — Pause/UnPause both slot sources (only if they have a clip/are playing). The crossfade loop (`RunTrackAsync`) and `MonitorTrackEndAsync` and `FadeOutAndStopAsync` each add, right after their `await`, `if (_owner.MusicPaused) continue;` (park without advancing `elapsed`/time checks while paused).
- **Audio façade:** add `PauseMusic`/`ResumeMusic`.
- **Tests:** update `EffectiveVolumes...` (music no longer volume-gated; instead `PushMute` → `_music` sources paused). Add: `PushMute_PausesMusic_PopResumes` (music source `isPlaying` false after PushMute, true after PopMute); `PauseMusic_ComposesWithAdsMute` (paused if either holds; resumes only when both clear); SFX mute still gates (existing loop test already covers). Gate.

### Task 2: FadeChannelVolume + Changed event

**Files:** `Runtime/AudioService.cs`, `Runtime/Audio.cs`, `Tests/Editor/AudioServiceTests.cs`.

- **Changed event:** `public event Action Changed;` raised at the end of `SetSetting` (only when the value actually changed — `SetSetting` already early-returns on no-op).
- **FadeChannelVolume:** `public void FadeChannelVolume(AudioChannel channel, float target, float seconds)` — clamp target 0..1; `seconds <= 0` sets the channel's `_musicFade`/`_sfxFade` immediately + `ApplyVolumes`; else a generation-guarded (`_musicFadeGen`/`_sfxFadeGen`) `async void` unscaled loop lerping the multiplier from current to target, calling `ApplyVolumes` each frame. Does NOT touch `MusicVolume`/`SfxVolume` (the durable settings) and does NOT raise `Changed`.
- **Audio façade:** `FadeChannelVolume` + `Changed` (event add/remove forwarding to the bound service, or documented as instance-only — mirror as `Audio.Changed` pass-through if a service is bound).
- **Tests:** `Changed` fires on a real setting change, not on a no-op write; `FadeChannelVolume(ch, 0.3f, 0f)` sets the multiplier instantly (assert via `MusicPlayVolume`/`EffectiveSfxVolume`) and leaves `MusicVolume`/`SfxVolume` untouched; `FadeChannelVolume` does not raise `Changed`. Gate.

### Task 3: Settings-mute = Stop + remembered request

**Files:** `Runtime/AudioService.cs`, `Tests/Editor/AudioServiceTests.cs`.

- **Requested intent:** `private enum RequestKind { None, TrackKey, TrackClip, Playlist }` + fields `_requestKind`, `_requestKey`, `_requestClip`, `_requestLoop`. `PlayMusic(key)`/`PlayMusic(clip)`/`PlayPlaylist(key)` record the request, then **only start the player if `MusicEnabled`**; `StopMusic()` clears the request + stops.
- **MusicEnabled setter:** on the change, after persist + `Changed`: if now false → `_music.Stop()` (keep the request); if now true → `StartRequested()` (replay from the top).
- **StartRequested():** dispatch on `_requestKind` to the existing `_music.PlayEntry`/`PlayClip`/`PlayPlaylist` (resolving the entry/playlist from the catalog as `PlayMusic`/`PlayPlaylist` do today).
- **Tests:** `MusicEnabled=false` stops (`ActiveMusicKey` null) but a later `MusicEnabled=true` replays the remembered key from the top; `PlayMusic` while disabled records but doesn't start (`ActiveMusicKey` null), enabling starts it; boot-muted (`saveSystem` with `music=false`) + `PlayMusic` → nothing plays until enabled; `StopMusic` clears the request so enabling plays nothing. Gate.
- [ ] Commit `feat(audio): settings mute stops music and replays the remembered request on re-enable`.

### Task 4: PreloadAsync (addressable music warm-up)

**Files:** `Runtime/AudioService.cs`, `Runtime/MusicPlayer.cs`, `Runtime/Audio.cs`, `Tests/Editor/AudioServiceTests.cs`.

- **Preload cache** (in `MusicPlayer` or a small helper owned by it): `Dictionary<string, AsyncOperationHandle<AudioClip>>`. `Awaitable PreloadAsync(AudioCatalog.Entry entry)` — if addressable and not already cached, `LoadAssetAsync`, await, cache the handle (release on failure). A slot that plays a preloaded key uses the cached clip and does NOT own/release its handle (the cache owns it); `RunTrackAsync`'s addressable branch checks the cache first. `Dispose` releases all cached handles.
- **AudioService:** `public async Awaitable PreloadAsync(string musicKey)` — resolve entry, delegate; no-op (warn) for non-addressable or unknown keys.
- **Audio façade:** `PreloadAsync`.
- **Tests (state-level; addressable load is play-mode verified):** `PreloadAsync` on a non-addressable entry no-ops without error; unknown key errors; direct-clip entries are unaffected. (The resident-reuse + release-on-Dispose paths are review + shikaku play-mode verified — headless has no addressables.)
- [ ] Commit `feat(audio): PreloadAsync warms addressable music clips`.
