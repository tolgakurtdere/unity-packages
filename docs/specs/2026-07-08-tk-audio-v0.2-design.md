# com.tk.audio 0.2.0 — Design (SFX control + editor authoring)

Approved by Tolgahan 2026-07-08 (scope split from the MasterAudio-teardown backlog: v0.2 = SFX + editor tools, v0.3 = music/settings polish). Evidence: g-brain used handle-based `PlaySound` ×46 (loop/stop patterns) and `[SoundGroup]` string fields ×1494 — the two highest-leverage gaps. Additive; existing 373 tests stay green; tag after game-shikaku verification (editor tools verified by authoring shikaku's catalog; loop-SFX by a light play-mode check).

## In scope

### Runtime — SFX
1. **`AudioHandle`** (public readonly struct) — the return type for looping / tracked SFX. Members: `bool IsPlaying`, `void Stop()`, `void FadeOutAndStop(float seconds)`. **Default-safe:** `default(AudioHandle)` and a handle to an already-stopped/reused voice are silent no-ops (kills the reference's two-level `ActingVariation` null-check chore by design). Backed by an internal owner reference + an int id; a stale id resolves to no-op.
2. **`AudioHandle PlaySfxLoop(string key)`** + **`AudioHandle PlaySfxLoop(AudioClip clip, float volumeScale = 1f, float pitch = 1f)`** — start a looping SFX and get a handle to stop it. Loops use **dedicated tracked sources**, NOT the one-shot pool: a loop's lifetime is unbounded, so mixing it with the auto-return pool would break the pool's return contract. A small loop-source list grows on demand and is returned on Stop.
3. **`void StopSfx(string key)` / `void StopAllSfx()`** — stop active one-shots and loops by key, or everything. (Loops by key: stops all loop voices started from that key.)
4. **`maxConcurrentVoices`** — new per-entry catalog field (default `0` = unlimited). When a key at its cap plays again, the **oldest** live voice of that key is culled first (MA's weight-limit half; complements the time-based retrigger throttle, which is a different axis).
5. **`void PlaySfx(string key, float volumeScale, float delaySeconds)`** overload — schedule the shot after an unscaled delay (the package-side answer to the reference doc's DOTween `AppendCallback` same-frame trap). The retrigger throttle is evaluated at call time, not after the delay.

### Editor — authoring (new `TK.Audio.Editor` asmdef)
6. **`[AudioKey]` / `[AudioPlaylistKey]`** attributes (in the runtime asmdef, so game code annotates `string` fields) + PropertyDrawers (in the editor asmdef). The drawer shows a dropdown of keys collected by **scanning every `AudioCatalog` asset in the project** (locked decision — right for the single-catalog norm; auto-covers the deferred multi-catalog case). A value not present in any catalog is shown with a warning affordance but never cleared.
7. **Catalog auto-populate** — a custom `AudioCatalog` inspector button "Add Entries From Selected Clips": from `AudioClip`s selected in the Project window, append entries with `key` = clip name, `channel` guessed (default Sfx), skipping keys already present. The clip→key derivation and dedupe are pure functions (unit-tested); the AssetDatabase/inspector wiring is review + shikaku-editor verified.

## Design contracts

- **Loop voices honor mute/volume.** `ApplyVolumes` sweeps loop sources too, so `PushMute` (ads) silences loops immediately and `SfxEnabled=false` stops spawning new ones (existing loops are volume-gated to 0, consistent with one-shot behavior — a running ambient shouldn't hard-cut on a settings toggle; the game calls `Stop()` when it wants it gone).
- **`FadeOutAndStop`** is generation-guarded per voice: a second fade or an external `Stop` during a fade is safe; the source returns to the loop pool exactly once.
- **Voice cull is per key.** `maxConcurrentVoices` counts only live voices sharing that key; direct-clip `PlaySfx(clip, …)` has no key and is never capped (documented, same class as the throttle bypass).
- **`StopAllSfx`** stops one-shots and loops but does not touch music.
- Editor drawers/tool are **editor-only** (`TK.Audio.Editor`, `includePlatforms: ["Editor"]`); no runtime cost, no new runtime dependency.

## Out of scope (v0.3 and beyond)
Mute redesign (ads=Pause / settings=Stop+intent), `PauseMusic`/`ResumeMusic`, `FadeChannelVolume`, settings `Changed` event, `PreloadAsync` — all v0.3. Still deferred to a real consumer: level-scoped additional catalogs, addressable SFX + warm window, ducking, AudioMixer, named categories.

## Testing
EditMode, harness: `AudioHandle` default no-op + Stop/IsPlaying state via an injected/inspected loop source; loop uses a dedicated source (host child count) and returns on Stop; `StopSfx(key)`/`StopAllSfx` clear the right voices; `maxConcurrentVoices` culls oldest at the cap (direct clips uncapped); `PlaySfx(delay)` doesn't spawn before the delay; mute sweeps loop volume. Editor: pure key-collection + clip→key/dedupe functions unit-tested; PropertyDrawer rendering and the populate button are review + shikaku-editor verified. Existing 373 stay green.
