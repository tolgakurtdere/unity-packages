# com.tk.audio 0.1.0 — Design

Approved by Tolgahan 2026-07-07 (brainstorm Q&A + MasterAudio teardown; playlists promoted into v1 by explicit request). No portable in-house reference exists — g-brain wraps DarkTonic MasterAudio (28k LOC, licensed, unpackageable); this is a fresh design that keeps MA's proven ideas (per-entry Addressables, playlists, retrigger throttling, clip variations) and fixes the pains its wrapper documented (the `ChangeMusicStatusByAdManager` latch marked "TODO: not a good method", music toggle restarting playlists and losing position, PlayerPrefs coupling).

## Decisions (locked)

1. **Shape:** plain-class `AudioService` (iap/ads pattern) constructed at the composition root and registered in `AppContext`; creates its own host GameObject (`[TK.Audio]`, `DontDestroyOnLoad` only when playing) carrying two music `AudioSource`s and an `ObjectPool<AudioSource>` for SFX. Static `Audio` façade (Analytics pattern): `Audio.Bind(service)`, mirrored calls, warn-once no-op when unbound, domain-reload-off static reset.
2. **Channels:** two built-in — `Music`, `Sfx` — each with `Enabled` + `Volume` (0–1). Named categories deliberately deferred; the state container must extend additively.
3. **Persistence: optional.** `AudioService(AudioCatalog catalog = null, ISaveSystem saveSystem = null, int sfxPoolSize = 8)`. With a save system the service loads on construction and persists every settings change (key `"tk_audio_settings"`); with null it is runtime-only and the game pushes values (shikaku mode — its `SettingsService` already owns the flags).
4. **Two-level mute** (the back-input pattern): user settings are durable and never written by the package; `PushMute()`/`PopMute()` is a ref-counted temporary suppression (`RefCountLock`; unbalanced pop throws). Effective channel gain = `Enabled && !IsMuted ? Volume : 0`. Ads wiring is one line of game glue on `AdsOptions.AudioMuteSetter` (`m => { if (m) a.PushMute(); else a.PopMute(); }`) — documented in the README.
5. **Catalog: optional SO** (`Create → TK/Audio Catalog`), string-keyed. Entry: `{ key, channel, AudioClip[] clips (random pick = variation-lite), AssetReferenceT<AudioClip> addressableClip, volumeScale = 1, pitchVariance (±, Sfx), minRetriggerInterval = 0.05 (Sfx) }`. Direct-clip API overloads work with no catalog at all. Unknown keys: error log + no-op.
6. **Clip sources:** an entry uses its direct `clips` when any are set, else its `addressableClip`. **Addressables is Music-oriented in v1**: music entries may be addressable (loaded on demand inside the generation-guarded pipeline; the handle is held per music source and released when that source drops the clip — abandoned mid-load releases immediately). An Sfx entry with only an addressable clip logs an error and is skipped (deferred: addressable SFX + MA-style delayed-release warm window). No new dependency cost: the package declares `com.unity.addressables` (2.8.1), which consumers already have via com.tk.core.
7. **Music engine:** two sources, crossfade (`MusicCrossfadeSeconds`, default 0.5; 0 = hard cut), unscaled-time fades, generation counter guarding every await (loads, fades, playlist advance) — latest request wins. `PlayMusic(key | clip, loop = true)` is idempotent when the same track is already active. `StopMusic()` fades out. **`MusicEnabled = false` gates volume only** — playback position (and playlist progression) continues silently, so re-enabling resumes where the player expects (fixes MA-wrapper's restart-on-toggle).
8. **Playlists (v1):** defined in the catalog as `{ key, entryKeys[] (Music entries of this catalog), shuffle, loop = true }`. `PlayPlaylist(key)`: shuffle order once per start (Fisher–Yates on a copy), play tracks non-looping, auto-advance with crossfade (advance starts when remaining time ≤ crossfade), at the end loop or stop. Idempotent when the same playlist is already playing. `ActiveMusicKey` exposes the current entry key (null for direct clips), `ActivePlaylistKey` the playlist (null in single-track mode).
9. **SFX engine:** pooled one-shots (`ObjectPool<AudioSource>`); per-entry pitch sampled as `1 ± pitchVariance`; source returned to the pool after `clip.length / pitch` via an unscaled frame loop. **Retrigger throttle**: a key played again within its `minRetriggerInterval` (unscaled clock) is silently dropped — the same-frame coin-spam guard MA solves with retrigger percentages. Disabled/muted channel → nothing is spawned. Direct-clip overload `PlaySfx(clip, volumeScale = 1, pitch = 1)` bypasses catalog tuning and throttling (documented). SFX is timeScale-independent (pause-menu clicks work; only our return timer runs unscaled).
10. **Rollout:** branch `audio-v0.1.0`; CHANGELOG `[0.1.0] - Unreleased`; harness gains the 9th package. Tag `com.tk.audio/0.1.0` only after play-mode verification in game-shikaku (wire SettingsService flags, a click SFX, menu music/playlist, ad-mute glue).

## Out of scope (ROADMAP)

Looping-SFX handles, ducking, AudioMixer integration, named categories, addressable SFX + delayed-release window, remote playlist config.

## Testing

EditMode, frame-free where possible: effective-gain math across enabled/volume/mute-stack combinations; unbalanced `PopMute` throws; persistence round-trip + null-save runtime-only mode (local `FakeSaveSystem`); catalog entry/playlist resolution, unknown keys, addressable-SFX rejection; pitch-variance bounds; retrigger throttle (monotonic clock injectable or `Time.unscaledTime`-tolerant assertions); SFX skip-when-disabled (no pool spawn); PlayMusic idempotence + `ActiveMusicKey`/`ActivePlaylistKey` state; shuffle produces a permutation. Fades/advance/addressable loading are play-mode verified in shikaku before tagging; existing 345 tests stay green.
